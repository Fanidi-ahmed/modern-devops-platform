# Bloc 00 — Fondations

## But du bloc

Préparer une base de travail propre pour un projet DevOps moderne.

## Ce que j'apprends

- organiser un projet sur le long terme
- séparer code, documentation et journal d'erreurs
- préparer Git correctement
- éviter le désordre dès le départ

## Idée clé

Une bonne architecture de dossiers réduit la confusion future.

## Ce que je dois retenir

- un repo propre aide autant à apprendre qu'à produire
- documenter les erreurs fait gagner beaucoup de temps
- la structure du projet reflète la structure de pensée

## Résultat attendu

Avoir un dépôt propre, lisible, prêt pour les blocs techniques suivants.
